4. VERIFIERING EFTER MERGE
================================================================================

Checklist (alla måste vara JA):

FRONTEND:
[ ] analysis.html renderar utan console-errors
[ ] Uppladdning av bild fungerar (drag-and-drop + klick)
[ ] Uppladdning av video extraherar frame
[ ] Webcam öppnas och visar live-video
[ ] Webcam-snapshot fungerar
[ ] Kamerabyte fungerar på mobil
[ ] "Analysera min hud"-knappen skickar till /api/analysis
[ ] Resultat visas med stagger-animation
[ ] Produktrekommendationer visas med korrekta bilder/priser
[ ] Chatt-input finns och skickar meddelanden
[ ] Chatt-bubblor renderas korrekt (användare vs terapeut)
[ ] "Skriver..."-indikator visas under väntan
[ ] Privacy-meddelande är synligt
[ ] Layout är tvådelad på desktop, en-kolumn på mobil
[ ] Inga bilder/data sparas i localStorage

BACKEND:
[ ] POST /api/analysis returnerar { content, responseId, usage }
[ ] POST /api/analysis/chat returnerar { content, responseId }
[ ] Rate limiting fungerar (429 efter gräns)
[ ] System prompt skickas INTE från klienten
[ ] Felmeddelanden är på svenska

DESIGN:
[ ] Typografi matchar designsystemet (SF Pro, --dark, --green)
[ ] Border-radius konsekvent (24px kort, 20px bubblor, 28px input)
[ ] Transitions mjuka (0.3s var(--ease))
[ ] Hover-effekter på knappar och interaktiva element
[ ] prefers-reduced-motion hanterad
[ ] Responsivt: mobil, tablet, desktop

PROMPT:
[ ] Analyssvaret innehåller sektioner: Hudanalys, Livsstil, Produkter, Undvik
[ ] Chatten svarar på svenska
[ ] Chatten håller sig till mikrobiom och ECS
[ ] Max 2-3 produkter rekommenderas
[ ] Inga emojis i svaren
[ ] Tonen är varm, inte klinisk
